# Story FE-5: 캐릭터 조회·상세·수정·삭제·활성 지정·도감

Status: draft

## Story

As a 사용자,
I want 보유 캐릭터 목록과 상세(레이더·리포트/퀴즈 2-페이저)를 보고 편집·삭제·활성 지정하며 도감을 둘러보고,
so that 내 캐릭터를 관리하고 수집 현황을 확인한다.

## 목적과 범위

- 캐릭터 상세 #4: 좌 스프라이트 + 우 5각형 레이더(알고리즘·CS·DB·네트워크·프레임워크),
  하단 독립 페이지네이션 2-리스트(①리포트 기록 ②퀴즈 기록, 섞지 않음).
- 도감 #3: 커버플로우, 컬러=수집/회색=미획득.
- 수정 가능 속성만 편집(능력치·전투력·진화 상태는 read-only, FR-5).
- 삭제 후 활성 캐릭터 단일성 유지(FR-6), 활성 지정 최대 1개(FR-7).

## 현재 구현 상태

- mock 기반 구현 존재.

## Acceptance Criteria

### AC1 — 목록·상세 조회
- **Given** 보유 캐릭터가 있는 사용자가,
- **When** 캐릭터 상세 `/character/:id`에 진입하면,
- **Then** 좌측 스프라이트(`CgSprite`)와 우측 5각형 레이더(`CgRadar`, `algo·cs·db·net·fw` / `STAT_LABELS`)가 표시되고, 하단에 **독립 페이지네이션 2-리스트**(① 리포트 기록 ② 퀴즈 기록)가 각각 별도 페이저로 표시된다(두 리스트를 섞지 않음).
- **And** 각 리스트가 비어 있으면 "아직 리포트가 없어요." / "제출한 퀴즈가 없어요." 빈 카피를 보여주고, 항목이 PAGE 단위를 넘으면 ‹/› 페이저가 나타난다(경계에서 비활성).

### AC2 — 편집(능력치/전투력/진화 read-only)
- **Given** 캐릭터 상세에서 수정 가능한 속성(이름·디자인 키워드·성격 등)을 편집할 때,
- **When** 사용자가 편집을 저장하면,
- **Then** 수정 가능한 속성만 갱신되고, **능력치(5스탯)·전투력·진화 상태는 read-only**로 사용자가 직접 바꿀 수 없다(FR-5; 성장 결과는 리포트/퀴즈로만 변동).
- **Note(현 구현 갭):** `CharacterDetailView`에 편집 컨트롤이 아직 없음 — 본 AC 충족을 위해 `vue/` 안에서 편집 UI를 보강해야 한다.

### AC3 — 삭제 + 활성 단일성
- **Given** 활성 캐릭터를 포함해 캐릭터를 보유한 사용자가,
- **When** 특정 캐릭터를 삭제하면,
- **Then** `deleteCharacter(id)`로 제거되고, 삭제 대상이 활성이었고 남은 캐릭터가 있으면 첫 번째 캐릭터가 자동 활성화되어 **활성 캐릭터 단일성**이 유지된다(FR-6). 캐릭터가 0개가 되면 대시보드 빈 상태로 연결된다(FE-3 AC2).
- **Note(현 구현 갭):** 삭제 컨트롤이 뷰에 미연결 — 보강 필요.

### AC4 — 활성 지정 최대 1
- **Given** 비활성 캐릭터를 보유한 사용자가,
- **When** 특정 캐릭터를 활성으로 지정하면,
- **Then** `setActive(id)`로 해당 캐릭터만 active가 되고 나머지는 모두 비활성화된다(동시 활성 최대 1개, FR-7).
- **Note(현 구현 갭):** 활성 지정 컨트롤이 뷰에 미연결 — 보강 필요.

### AC5 — 도감 수집/미획득 표현
- **Given** 사용자가 도감 `/codex`에 진입했을 때,
- **When** `codex()`가 보유 + 잠금 항목을 반환하면,
- **Then** 보유 캐릭터는 컬러로, 미획득(`owned:false`, 이름 `???`)은 회색으로 구분 표시한다(색만이 아니라 이름/상태로도 구분 가능해야 함, FE-1 접근성 기준).

## 검증

- 상세 진입 시 레이더 5스탯과 두 리스트가 독립 페이저로 동작하는지 확인.
- 편집 저장 시 능력치/전투력/진화가 read-only로 유지되는지 확인.
- 활성 캐릭터 삭제·전환 시 항상 활성 1개 불변식이 유지되는지 확인.
- 도감 수집/미획득 시각 구분 확인, `npm run build` 통과.
- 편집·삭제·활성 컨트롤 보강은 모두 `vue/` 하위 파일로 한정.

# Character Image 1. Gemini Sprite Generation Core and Local Storage

## Status

done

## Story

As a FastAPI AI 서버 개발자,
I want a core sprite generation service that injects a sanitized design keyword into the canonical prompt, calls an injectable Gemini image client, validates transparent PNG bytes, and saves the result to local runtime storage,
so that Commitgotchi character image generation can be implemented safely before the FastAPI endpoint and future S3 storage are added.

## Goal

사용자 디자인 키워드 또는 디자인 prompt 후보를 받아 canonical prompt template에 안전하게 주입하고, Gemini image model로 2x3 transparent PNG sprite sheet를 생성한 뒤 FastAPI 내부 local directory에 저장하는 core service를 설계한다.

이번 story는 endpoint를 만들지 않는다. Spring Boot 호출 계약과 HTTP schema는 Story 2에서 다룬다. S3-ready storage adapter formalization은 Story 3에서 다룬다.

## Background

PRD FR-3은 캐릭터 생성 직후 AI가 6프레임 스프라이트시트 이미지를 생성하고, 실패 시 기본 스프라이트 세트를 사용한다고 정의한다. Architecture §7.6은 단일 이미지가 아니라 진화 2단계 × 감정 3종의 2x3 투명 PNG 스프라이트시트를 요구한다.

현재 S3 bucket은 준비되어 있지 않다. 따라서 이 story의 완료 기준은 Gemini image generation result를 local runtime directory에 저장하는 core service다.

중요한 계약:

- FastAPI는 캐릭터 record를 생성하거나 저장하지 않는다.
- FastAPI는 Spring Boot DB에 직접 접근하지 않는다.
- 이번 story는 `POST /api/ai/commitgotchi` endpoint를 만들지 않는다.
- 실제 Gemini 호출 테스트와 실제 S3 호출 테스트는 금지한다.
- 실패 시 caller가 fallback sprite를 선택할 수 있는 safe fallback result를 반환한다.

## Scope

이번 story에서 구현할 것은 core generation + local save까지다.

1. Config
   - Gemini image model name을 env/config에서 읽는다.
   - timeout/retry limit/output directory 같은 값도 env/config로 관리한다.
   - 기본 model 후보는 배포 시점에 확인된 image generation 가능한 작은/빠른 Gemini 계열 모델로 둔다.
   - 테스트는 model name이 실제 호출에 사용되지 않도록 fake client를 주입한다.

2. Prompt template
   - 아래 canonical prompt template을 FastAPI image module에 둔다.
   - `designKeyword`만 template placeholder에 주입한다.
   - raw prompt 전체를 logs/errors에 남기지 않는다.

```text
Create a transparent PNG sprite sheet for a retro handheld virtual pet game.

Six distinct pixel-art creatures arranged in a precise 2x3 layout, no grid lines and no text.
Each creature is drawn as a crisp pixel-art sprite on a logical pixel grid, enlarged for extraction with sharp nearest-neighbor edges.

Columns: happy, sad, angry.
Row 1: baby stage, logical 16x16 sprite design.
Row 2: mature evolved stage, logical 18x18 sprite design.

All six sprites share the same creature identity based on this design keyword: "{designKeyword}".
Use clean black outlines, vibrant limited palette, simple readable silhouette, front-facing pose, transparent background, no shadows, no UI, no labels.
The output should function as a game asset sprite sheet with alpha transparency.
```

3. `designKeyword` sanitization
   - `None`, 빈 문자열, whitespace-only 입력을 reject하거나 safe default/fallback으로 분류한다.
   - length cap을 둔다.
   - control characters, null byte, line break abuse를 제거하거나 정규화한다.
   - prompt injection 문구가 들어와도 template instruction이 깨지지 않도록 plain text로 escape/quote한다.
   - sanitized keyword와 loggable summary를 분리한다.

4. Image generation client abstraction
   - fake 주입 가능한 protocol/interface를 둔다.
   - production client는 Gemini SDK 호출 세부사항을 캡슐화한다.
   - core service는 SDK response object를 직접 알지 않고 PNG bytes만 받는다.
   - generation timeout/retry failure는 typed error 또는 fallback classification으로 변환한다.

5. PNG validation
   - generated bytes는 non-empty PNG bytes여야 한다.
   - PNG signature를 확인한다.
   - dimensions를 추출한다.
   - alpha channel 존재 여부를 확인한다.
   - 2x3 layout 자체를 픽셀 단위로 완벽히 판정하지는 않는다. 다만 dimensions와 metadata가 프레임 분할에 사용할 수 있는 정상 범위인지 확인한다.

6. Local storage
   - 저장 root는 repo 내부 runtime/data 경로로 둔다.
   - 권장 root: `fastapi/runtime/data/character-images/`
   - generated PNG가 git에 들어가지 않도록 `.gitignore` 또는 문서 guardrail을 둔다.
   - filename은 `imageRequestId`, server-generated UUID, content hash 등 안전한 값에서 만든다.
   - `designKeyword`를 filename/path segment에 직접 쓰지 않는다.
   - local storage write는 configured root 밖으로 나가지 않아야 한다.

7. Result shape
   - core service는 endpoint와 독립적인 result object를 반환한다.
   - success는 local path, metadata, generated flag를 가진다.
   - failure는 fallback classification과 safe reason code를 가진다.

권장 result shape:

```json
{
  "imageStatus": "READY",
  "storageKind": "LOCAL",
  "localPath": "fastapi/runtime/data/character-images/users/1/image-request-uuid.png",
  "contentType": "image/png",
  "spriteMeta": {
    "columns": 3,
    "rows": 2,
    "frameMap": {
      "baby": { "happy": [0, 0], "sad": [0, 1], "angry": [0, 2] },
      "mature": { "happy": [1, 0], "sad": [1, 1], "angry": [1, 2] }
    },
    "frame": { "babyPx": 16, "maturePx": 18 },
    "transparent": true
  }
}
```

Fallback result 예:

```json
{
  "imageStatus": "FALLBACK",
  "storageKind": "NONE",
  "fallbackReason": "IMAGE_GENERATION_FAILED",
  "spriteMeta": null
}
```

## Acceptance Criteria

1. `designKeyword`를 canonical prompt template의 `{designKeyword}` placeholder에만 주입한다.
2. `designKeyword` sanitization은 빈 값, 길이 초과, control character, line break abuse를 안전하게 처리한다.
3. sanitized keyword는 prompt용 문자열과 loggable summary를 분리한다.
4. final prompt 원문 전체는 logs/errors/test snapshots에 남기지 않는다.
5. Gemini image model 설정은 env/config로 관리된다.
6. model name 기본 후보는 image generation 가능한 작은/빠른 Gemini 계열 모델을 가리키되, 테스트에서 실제 호출되지 않는다.
7. image generation client는 protocol/interface 또는 동등한 abstraction으로 fake 주입 가능하다.
8. core service는 SDK-specific response object가 아니라 PNG bytes를 입력으로 받아 후속 validation/storage를 수행한다.
9. generation timeout, SDK error, empty response는 fallback classification으로 변환된다.
10. 생성 결과는 `bytes` 또는 동등 binary type으로 처리한다.
11. PNG validation은 PNG signature를 확인한다.
12. PNG validation은 non-empty bytes를 확인한다.
13. PNG validation은 width/height dimensions를 확인한다.
14. PNG validation은 alpha channel 존재 여부를 확인한다.
15. invalid PNG, alpha 없는 PNG, dimensions 추출 실패는 저장하지 않고 fallback result로 분류한다.
16. local 저장 경로는 repo 내부 `fastapi/runtime/data/character-images/` 또는 동등 runtime/data 경로 아래로 제한된다.
17. storage layer는 path traversal이 불가능하도록 normalized path가 storage root 안에 있는지 확인한다.
18. 저장 filename/object key는 user input을 직접 사용하지 않는다.
19. 저장 성공 시 result에는 local path, content type `image/png`, sprite metadata가 포함된다.
20. 저장 실패 시 safe fallback result를 반환한다.
21. fallback result는 캐릭터 생성 흐름을 막지 않는 `imageStatus="FALLBACK"` 또는 동등 classification을 가진다.
22. logs/errors에 secret, API key, generated image bytes, full prompt가 노출되지 않는다.
23. tests는 fake image client와 fake storage만 사용한다.
24. 실제 Gemini API 호출을 하는 unit/integration test는 없다.
25. 실제 S3 호출 또는 S3 config requirement는 없다.
26. FastAPI endpoint, Spring Boot DB 접근, report/quiz flow 변경은 추가하지 않는다.

## Test Requirements

필수 테스트는 fake image client와 fake storage를 사용한다.

- sanitized `designKeyword`가 prompt template에 정확히 한 번 주입되는지 검증한다.

- prompt builder가 full prompt를 loggable value로 반환하지 않는지 검증한다.
- fake image client가 PNG bytes를 반환하면 core service가 validation과 storage를 호출하는지 검증한다.
- fake image client가 timeout/error/empty bytes를 반환하면 fallback result가 나오는지 검증한다.
- valid transparent PNG fixture bytes가 accepted 되는지 검증한다.
- invalid signature bytes가 rejected 되는지 검증한다.
- alpha channel 없는 PNG fixture bytes가 rejected 되는지 검증한다.
- dimensions가 0이거나 추출 불가능한 bytes가 rejected 되는지 검증한다.
- local storage path가 configured root 밖으로 나가지 않는지 검증한다.
- filename generation이 `designKeyword`를 직접 포함하지 않는지 검증한다.
- storage failure가 fallback result로 분류되는지 검증한다.
- logs/errors/test snapshots에 API key, full prompt, image bytes가 없는지 검증한다.

### PNG Validation Note

`fastapi/requirements.txt`에 image parser dependency가 없다. 구현자는 다음 중 하나를 선택한다.

- PNG signature/IHDR/color type을 직접 읽는 작은 validator를 구현한다.
- Pillow 같은 dependency를 도입한다면 `requirements.txt`와 tests에 명시하고, dependency 도입 이유를 남긴다.

MVP validation은 "AI가 정확한 2x3 pixel-art layout을 그렸는지"를 시각적으로 판정하지 않는다. PNG artifact가 투명 PNG이며 프레임 분할에 필요한 dimensions가 있는지만 확인한다.

### Implementation Plan

- FastAPI 전역 `Settings`에 Gemini image model, API key, timeout, retry, local storage root를 추가하고 image 전용 settings adapter에서 storage root를 `fastapi/runtime/data/character-images` 하위로 제한했다.
- `app.image` 패키지를 새로 만들고 prompt builder/sanitizer, Gemini client protocol 및 production wrapper, PNG validator, local storage, core service/result model을 분리했다.
- service 테스트는 fake image client/fake storage를 주입하고, PNG 검증 테스트는 테스트 코드에서 생성한 tiny PNG bytes만 사용했다.

### Completion Notes

- Gemini image model 기본값은 `GEMINI_IMAGE_MODEL` env/config의 default `gemini-2.5-flash-image`로 구현했다.
- prompt template에는 sanitized `designKeyword`만 1회 주입하고, prompt/result repr 및 loggable summary에는 full prompt를 담지 않도록 했다.
- invalid keyword, generation timeout/error/empty bytes, invalid PNG, missing alpha, local storage failure는 safe fallback result로 변환된다.
- PNG validator는 dependency 추가 없이 PNG signature, chunk CRC, IDAT payload, IHDR dimensions, frame-splittable size, alpha channel 여부를 직접 확인한다.
- design keyword sanitizer는 null/control/line break abuse와 zero-width/bidi format control을 제거하거나 정규화한다.
- local storage는 `fastapi/runtime/data/character-images/` 하위로 제한되며 filename은 user input 원문 대신 hash/UUID 기반으로 생성된다.
- generated image runtime directory는 `fastapi/.gitignore`에서 보호한다.
- 이번 story에서는 FastAPI endpoint, Spring Boot DB 접근, report/quiz/integration flow를 변경하지 않았다.

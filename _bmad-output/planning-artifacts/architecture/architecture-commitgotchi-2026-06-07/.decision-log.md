# Decision Log — commitgotchi Architecture

> 아키텍처 결정의 변경 이유와 범위를 시간순으로 기록한다. 전체 결정 인덱스는 `architecture.md` §9를 따른다.

## 2026-06-11 — Spring Boot 인증·인가 구현 증분 업데이트

- **업데이트 목적:** 전체 MVP에 앞서 Spring Boot 인증·인가를 바로 구현할 수 있도록 PRD FR-1~2, FR-24~28의 아키텍처 계약을 구체화했다.
- **입력 문서:** 최신 PRD `prd.md`, `addendum.md`, `.decision-log.md`를 확인했다.
- **범위 결정 AD-15:** 첫 구현 증분은 `users`, `refresh_tokens`와 인증 API/최소 검증 API로 제한한다. 전체 도메인 테이블과 Spring AI 기능은 제외한다.
- **보안 결정 AD-16~19:** 무상태 Spring Security Bearer JWT, HS256 Access Token 15분, opaque Refresh Token 30일, SHA-256 해시 저장, Rotation, 폐기 토큰 재사용 시 사용자 활성 Refresh Token 전체 폐기를 확정했다.
- **데이터 결정 AD-20~21:** 이메일은 trim+lowercase 정규화 후 DB CHECK+UNIQUE로 강제하고, Role은 `varchar(20)` + CHECK(`USER`, `ADMIN`)로 저장한다.
- **운영 결정 AD-22:** 초기 ADMIN은 공개 API나 Flyway Secret 시드가 아니라 제한된 운영 DB/CLI 절차로 프로비저닝한다.
- **오류/암호 결정 AD-23~24:** EntryPoint·DeniedHandler·ControllerAdvice가 동일 오류 계약을 사용하고, 비밀번호는 BCrypt cost 12를 사용한다.
- **마이그레이션 결정 AD-25:** 인증 증분의 스키마 변경은 Flyway `V1 users` → `V2 refresh_tokens` 순서로 수행하며 Hibernate는 `ddl-auto=validate`를 사용한다.
- **구조 정정:** 기존 문서의 Spring Boot 경로 `backend-spring/`을 실제 저장소 경로 `springboot/`로 정정했다.
- **충돌 기록:** 최신 PRD/Addendum의 퀴즈 동기 채점·이미지 비동기 큐와 기존 아키텍처의 퀴즈 웹훅·이미지 동기 HTTP가 충돌한다. 인증 증분 밖이므로 이번 업데이트에서 임의 해결하지 않고 후속 정합화 항목으로 남겼다.
- **미확정 사항:** Refresh Token 폐기/만료 레코드 7일 보존과 비밀번호 12~64자·UTF-8 72바이트 이하 정책은 `[ASSUMPTION]`으로 남겼다.

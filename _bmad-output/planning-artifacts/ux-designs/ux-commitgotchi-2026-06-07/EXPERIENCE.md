# Commit-Gotchi — Experience Spine

> **기존 Vue 목업(`docs/Commit-Gotchi 목업 (Vue) - 단독.html`)에서 증류했다. 목업이 IA·화면 구성의 기준이다.** 동작·플로우는 이 문서가, 시각 토큰은 `DESIGN.md`가 소유한다. `[ASSUMPTION]`은 PRD §8 미해결 질문 및 목업↔PRD 불일치와 연동.

## Foundation

데스크톱 웹 우선, 라이트(cozy) 테마 기본. 명명된 UI 시스템 없음 — `DESIGN.md`의 자체 Galmuri 픽셀 디자인 시스템(테마 cozy/device/cli 전환)을 따른다. 인증은 JWT(FR-1~2). 핵심 정신 모델: **학습 = 캐릭터 육아.** 활성 캐릭터 1개가 모든 학습 점수의 수신처(FR-7)이며, 화면은 이 캐릭터의 성장(육아점수·능력치·감정·진화)을 중심으로 돈다.

시간 구조(개정): **퀴즈는 풀이 즉시 접수 후 웹훅 결과 표시**(흐름 B) — 오전 9시 도착한 추천 퀴즈를 그 자리에서 풀고 바로 피드백·점수 반영을 본다. **학습 리포트는 자정 배치**(흐름 A, 비동기) → 다음날 오전 9시 일일 레포트·다음 추천 퀴즈 도착. **캐릭터 이미지는 캐릭터 생성 요청 안에서 Spring Boot가 FastAPI `POST /api/ai/commitgotchi`를 호출하는 동기 HTTP 흐름**(흐름 C). 즉 즉각 피드백(퀴즈/이미지)과 "오늘 심고 내일 수확"(리포트)이 공존하며, UI는 두 리듬을 구분해 다룬다.

> **용어 불일치(목업 ↔ PRD) — PM 확정 필요:** 목업은 주 지표를 **"육아점수"** 로 표기한다(랭킹 기준 = 육아점수). PRD Glossary는 **"전투력"**(=능력치 5종 총합, 1,000점 진화)을 쓴다. 본 스파인은 **UI 노출 라벨을 목업의 "육아점수"로** 채택하고, PRD의 전투력 정의(총합·진화 임계)는 그대로 계승한다(= "육아점수"가 곧 능력치 총합). 다운스트림에서 한 용어로 통일할 것. [ASSUMPTION]

## Voice and Tone

마이크로카피. 브랜드 보이스·미감은 `DESIGN.md`. 핵심: **시스템 카피는 담백·정확, 캐릭터 카피는 활성 캐릭터 personality 반영**(FR-23). 목업 톤은 친근한 구어체(예: 학습 기록 제목 "다익스트라 드디어 이해함!").

| Do | Don't |
|---|---|
| (시스템) "리포트 저장됨" · "제출 완료" | "🎉 성공적으로 저장되었습니다!" |
| (캐릭터, 기쁨) personality 반영 한 줄 | 성격 무관한 일반 격려 |
| (대기, 리포트) "오늘 학습 리포트는 자정에 분석돼요. 내일 오전 9시 도착." | "처리 중..." |
| (즉시, 퀴즈) "채점 중…" → 곧바로 점수·피드백 표시 | 퀴즈에 "자정에 채점" 안내(흐름 혼동) |
| (Fallback) "AI가 잠깐 쉬는 중 — 학습은 저장됐어요." | "Error" |
| 지표 라벨 "육아점수"로 통일 | "전투력/육아점수" 혼용 |

## Interaction Primitives

- 클릭으로 액션. 버튼은 hover 떠오름 / active 눌림(`DESIGN.md.Elevation`).
- 저장·제출은 명시적 버튼(의도적 커밋의 정서).
- **퀴즈 점수 반영·감정 갱신은 제출 즉시** 애니메이션으로 재생(흐름 B). **리포트 점수 반영·진화 연출은 결과 도착 시점(아침)** 에 재생(흐름 A).
- 캐릭터 상세 공부기록은 **리포트/퀴즈 2개 페이저를 분리** 운용. 도감은 커버플로우 좌우 탐색, 게시판/랭킹은 페이지네이션.
- 파괴적 액션(삭제)은 확인 1단계.
- **금지:** 캐러셀 남용, 콜드 진입 과한 히어로 애니메이션, 푸시/실시간 알림(PRD §5). 단, **퀴즈는 즉시 채점이 진실**이므로 즉답 인터랙션을 권장(리포트만 자정 배치).

## Accessibility Floor

행동 규칙. 시각 대비는 `DESIGN.md`. **픽셀 폰트 가독성 주의:**

- 긴 본문(리포트·피드백·퀴즈 해설)은 충분한 크기·줄간격의 Galmuri11/Gothic A1으로. 작은 메타만 Galmuri7/9.
- 감정·상태·도감 수집 여부는 색만으로 전달 금지 — 색 + 얼굴/라벨/잠금 아이콘 병기.
- 육아점수·스탯 수치는 게이지/레이더 외에 숫자 텍스트로도 제공.
- 클릭 타깃 ≥ 40×40px. 포커스는 읽기 순서. Reduce Motion 시 bob·confetti·firework는 정적 대체.
- cli 테마(네온 on 다크) 대비 확인. 키보드만으로 핵심 루프(리포트 작성→저장→제출) 완주.

## Key Flows

### Flow 1 — 윤석, 퇴근 후 오늘 공부를 캐릭터에게 먹인다 (핵심 루프, UJ-1)

1. 로그인된 대시보드 진입 — 히어로 스테이지에 활성 캐릭터(스프라이트) idle, 사이드 레일에 육아점수 게이지·감정·상태 메시지.
2. (오전 9시 도착한) **오늘의 추천 퀴즈 풀이·제출 → 즉시 채점**: "채점 중…" 후 점수·피드백이 그 자리에서 뜨고, 알고리즘 +1이 즉시 반영되며 게이지가 오르고 감정이 갱신된다(흐름 B). 당일 자정 전 재제출 가능.
3. "오늘의 학습 기록하기" → 일일 리포트 작성(#5): 기분 선택 + 제목 "Spring JPA N+1 문제와 해결법" + 내용 + 태그(알고리즘) 작성. 캐릭터 반응 미리보기 확인 후 **저장**.
4. 대시보드 "오늘 리포트는 자정에 분석돼요. 내일 오전 9시 도착" pending — 흐름 마감.
5. **(다음날 아침)** 재접속, "어제의 레포트 도착".
6. **Climax:** 일일 레포트에서 학습 분석·피드백·점수 변화량(알고리즘 +3, 네트워크 +1)·어제 퀴즈 종합 코멘트를 보고, 레이더/스탯 타일이 차오르며 육아점수 게이지가 오르고, 감정이 "기쁨"으로 바뀌며 personality 말풍선이 뜬다. 새 추천 퀴즈도 도착.
7. **Resolution:** 다음 학습 추천을 보고 닫는다.
8. **Fallback:** 리포트 실패 시 점수 0 + "AI가 잠깐 쉬는 중 — 학습은 저장됨"; 퀴즈 채점 실패 시 답안 저장 + "잠시 후 재시도" — 어느 쪽도 흐름 유지(FR-16, SM-3).

### Flow 2 — 지민, 첫 캐릭터를 만든다 (UJ-2)

1. 가입·로그인 직후 대시보드 빈 상태 "첫 분신 만들기".
2. 캐릭터 생성 폼: 이름 + **디자인 키워드** + 성격("칭찬 많지만 틀린 건 명확히 짚는") 입력 → 생성. 캐릭터 생성 요청 중 Spring Boot가 FastAPI 이미지 생성을 호출하고, 실패 시 기본 세트를 사용한다(흐름 C, FR-16).
3. **Climax (#11 생성 완료):** 이미지 PENDING 동안 로딩 플레이스홀더 → READY 시 완성된 1x3 스프라이트 캐릭터가 컨페티·불꽃과 함께 등장, 5스탯 0에서 시작.
4. **Resolution:** 대시보드로 돌아와 첫 리포트를 쓸 준비(→ Flow 1).

### Flow 3 — 윤석, 랭킹·도감·게시판을 둘러본다 (UJ-3 + 수집, 보조)

1. 랭킹(#7): 육아점수 포디움 TOP3 + 내 순위 하이라이트.
2. 도감(#3): 수집한 캐릭터는 컬러, 미획득은 회색 — 컬렉션 동기.
3. 공유 게시판(#8) → 공유 캐릭터 상세(#9)에서 평점·리뷰 열람·작성.
4. **Climax:** 자기 캐릭터를 게시판에 공유해 리뷰를 받는다.
5. **Resolution:** 비교·수집·경쟁이 다음 학습 동기로 이어진다.
